<p>假设有打乱顺序的一群人站成一个队列，数组 <code>people</code> 表示队列中一些人的属性（不一定按顺序）。每个 <code>people[i] = [h<sub>i</sub>, k<sub>i</sub>]</code> 表示第 <code>i</code> 个人的身高为 <code>h<sub>i</sub></code> ，前面 <strong>正好</strong> 有 <code>k<sub>i</sub></code><sub> </sub>个身高大于或等于 <code>h<sub>i</sub></code> 的人。</p>

<p>请你重新构造并返回输入数组&nbsp;<code>people</code> 所表示的队列。返回的队列应该格式化为数组 <code>queue</code> ，其中 <code>queue[j] = [h<sub>j</sub>, k<sub>j</sub>]</code> 是队列中第 <code>j</code> 个人的属性（<code>queue[0]</code> 是排在队列前面的人）。</p>

<p>&nbsp;</p>

<ul> 
</ul>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong>people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
<strong>输出：</strong>[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
<strong>解释：</strong>
编号为 0 的人身高为 5 ，没有身高更高或者相同的人排在他前面。
编号为 1 的人身高为 7 ，没有身高更高或者相同的人排在他前面。
编号为 2 的人身高为 5 ，有 2 个身高更高或者相同的人排在他前面，即编号为 0 和 1 的人。
编号为 3 的人身高为 6 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
编号为 4 的人身高为 4 ，有 4 个身高更高或者相同的人排在他前面，即编号为 0、1、2、3 的人。
编号为 5 的人身高为 7 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
因此 [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]] 是重新构造后的队列。
</pre>

<pre>
<strong>输入：</strong>people = [[6,0],[5,0],[4,0],[3,2],[2,2],[1,4]]
<strong>输出：</strong>[[4,0],[5,0],[2,2],[3,2],[1,4],[6,0]]
</pre>

## 基本思路

我的思维过程：

1、这种有两个维度的数据，大概率需要对某一个维度进行排序，否则的话非常难处理。

2、对哪个维度排序呢，关键要看排序后，**能否获取到更多的确定性**，从而找到解题的突破口。

我第一个尝试是对 `k` 进行从小到大排序，若 `k` 相同，则对 `h` 进行从小到大排序：

```java
Arrays.sort(people, (a, b) -> {
    if (a[1] == b[1]) {
        return a[0] - b[0];
    }
    return a[1] - b[1];
});
```

我的理由是，这样排好序后的第一个人，必然是 `queue` 中排第一个的人。我也许能够通过这个确定性，逐步推导后面的人的顺序。

但是很快发现一个问题，这个思路按照 `k` 从小到大排序，就是假设 `k` 小的一定排在靠前的位置。然而这个假设并不成立，`k` 小有两种情况：可能因为这个人真的靠前，所以 `k` 小；也可能因为这个人很高（`h` 较大）导致 `k` 小。

所以这个思路行不通。

那么按照身高 `h` 排序呢，能获得什么确定性信息？

我的想法是这样，这个 `k` 记录了这个元素前面比它大的元素个数，那么如果我按照 `h` 从大到小排序，在根据 `k` 进行修正，似乎可以得到正确的结果。

比方说，这是按照身高排序的结果：

```
[7, 7, 6, 6, 5, 4, 3]
```

假设身高为 5 的这个人前方有 4 个比他高的，但是他对应的 `k` 是 2，那么我就根据 `k` 把他往前移动到正确位置：

```
[7, 7, 5, 6, 6, 4, 3]
```

向前移动的过程中，会不会影响前面人的 `k` 值？

分两种情况：

1、如果前面人的身高大于 5，那么不会影响；

2、如果前面人的升高等于 5，那么会影响。不过我们可以在排序时，对身高相同的人按照 `k` 从小到大排序，这样就能保证不会出现这种情况了。

综上，这个思路是可行的：

我们先按照身高从大到小排序，如果身高相同，按照 `k` 从小到大排序。然后调用 [插入排序](https://labuladong.online/algo/data-structure-basic/insertion-sort/)，稍微修改一下标准插入排序的比较逻辑，就可以完成这道题。

这个算法的主要逻辑是两次排序，时间复杂度是 `O(N^2 + NlogN) = O(n^2)`，因为插入排序的最坏时间复杂度是 `O(n^2)`。

拓展一下，如果用一个 `O(NlogN)` 的稳定排序算法，而不是用 `O(N^2)` 的插入排序，这个算法的时间复杂度就降到 `O(NlogN)` 了，这个优化就留给你吧。

class Solution {
public:
    vector<vector<int>> reconstructQueue(vector<vector<int>>& people) {
        // 按照身高从高到低排序，如果身高相同，按照 k 从小到大排序
        sort(people.begin(), people.end(), [](const vector<int>& a, const vector<int>& b) {
            if (a[0] == b[0]) {
                return a[1] < b[1];
            }
            return a[0] > b[0];
        });

        insertSort(people);
        return people;
    }

    // 对标准插入排序的比较逻辑稍作修改
    // https://labuladong.online/algo/data-structure-basic/insertion-sort/
    void insertSort(vector<vector<int>>& people) {
        int n = people.size();
        int sortedIndex = 0;
        while (sortedIndex < n) {
            for (int i = sortedIndex; i > 0; i--) {
                // 我们将第 i 个人向前移动
                // 现在这个人前面有 i（不是 i-1，因为索引从 0 开始）个比他高的人
                // 他的目标是前面有 k 个比他高的人，所以把他往前移动
                int k = people[i][1];
                if (k < i) {
                    // swap(nums[i], nums[i - 1])
                    vector<int> tmp = people[i];
                    people[i] = people[i - 1];
                    people[i - 1] = tmp;
                } else {
                    break;
                }
            }
            sortedIndex++;
        }
    }
};

class Solution:
    def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
        # 按照身高从高到低排序，如果身高相同，按照 k 从小到大排序
        people.sort(key=lambda x: (-x[0], x[1]))

        self.insertSort(people)
        return people

    # 对标准插入排序的比较逻辑稍作修改
    # https://labuladong.online/algo/data-structure-basic/insertion-sort/
    def insertSort(self, people: List[List[int]]):
        n = len(people)
        sorted_index = 0
        while sorted_index < n:
            for i in range(sorted_index, 0, -1):
                # 我们将第 i 个人向前移动
                # 现在这个人前面有 i（不是 i-1，因为索引从 0 开始）个比他高的人
                # 他的目标是前面有 k 个比他高的人，所以把他往前移动
                k = people[i][1]
                if k < i:
                    # swap(nums[i], nums[i - 1])
                    people[i], people[i - 1] = people[i - 1], people[i]
                else:
                    break
            sorted_index += 1
